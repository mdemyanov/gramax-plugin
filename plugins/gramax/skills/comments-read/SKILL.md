# Gramax Comments Read

Чтение комментариев из Gramax-документов — вывод структурированного отчёта или краткой сводки с фильтрами.

## Триггеры

- "покажи комментарии"
- "прочитай комментарии к документу"
- "какие замечания есть"
- "что написали рецензенты"
- "список комментариев"
- "comments"

## Использование

```
/gramax:comments-read <путь_к_каталогу_или_md_файлу>
```

Поддерживаемые пути:
- Отдельный `.md` файл: `path/to/catalog/section/_index.md`
- Каталог (рекурсивно): `path/to/catalog/`

## Формат комментариев Gramax

### Inline-теги в `.md`

```markdown
Текст с <comment id="abc12">привязанным фрагментом</comment> продолжение.
```

- `<comment id="ID">` — ID (5 символов `[a-zA-Z0-9]`)
- Текст между тегами — фрагмент документа, к которому привязан комментарий

### Block-формат

```markdown
[comment:abc12]

Блок текста с комментарием.

[/comment]
```

### `.comments.yaml` (рядом с md)

Именование: `{basename}.comments.yaml` (например, `_index.md` → `_index.comments.yaml`).

```yaml
GaRkQ:
  comment:
    dateTime: '2026-04-01T10:00:00.000Z'
    user:
      mail: author@example.com
      name: Имя Автора
    content: 'Текст комментария'
  answers:
    - user:
        mail: reviewer@example.com
        name: Имя Рецензента
      dateTime: '2026-04-01T11:00:00.000Z'
      content: >-
        Многострочный текст ответа
        на комментарий
```

Варианты content: короткий в одинарных кавычках или `>-` для многострочного.
Варианты answers: `[]` (пустой массив) или массив с элементами.

## Workflow

```
1. FIND       → Найти md-файл(ы) и соответствующие .comments.yaml
2. PARSE-YAML → Прочитать все комментарии из .comments.yaml
3. MATCH-MD   → Найти в md-файле теги <comment id="..."> и [comment:id] — извлечь привязанный текст
4. REPORT     → Сформировать удобный отчёт
```

## Исполнение

Основной инструмент — `parse_comments.py`. Запускается через `uv run`:

### Вариант 1: отчёт (по умолчанию)

```bash
uv run ${CLAUDE_PLUGIN_ROOT}/scripts/parse_comments.py <path>
```

Выводит комментарии в удобочитаемом формате:

```
## Комментарии к: <file>

### [1] ID: abc12 | 2026-04-01T10:00:00.000Z | Alice
**Привязка в тексте:** "привязанным фрагментом"
**Комментарий:** Первый комментарий

**Ответы:**
  1. 2026-04-01T11:00:00.000Z | Bob: Ответ на первый комментарий

---
```

### Вариант 2: JSON

```bash
uv run ${CLAUDE_PLUGIN_ROOT}/scripts/parse_comments.py <path> --format json
```

Для программной обработки — массив объектов `{file, id, anchor, comment, answers, status}`.

### Вариант 3: сводка

```bash
uv run ${CLAUDE_PLUGIN_ROOT}/scripts/parse_comments.py <path> --summary
```

```
## Сводка по комментариям
- Всего комментариев: N
- С ответами: N
- Без ответов: N
- Авторы:
  - Имя: количество
- Период: {первая дата} — {последняя дата}
```

## Фильтры

| Флаг | Действие |
|------|----------|
| `--author NAME` | Только комментарии конкретного автора (совпадает в комментарии или в ответах) |
| `--unanswered` | Только без ответов |
| `--answered` | Только с ответами |

Комбинируются. Например: только неотвеченные замечания от Alice:

```bash
uv run ${CLAUDE_PLUGIN_ROOT}/scripts/parse_comments.py <path> --author Alice --unanswered
```

## Особенности

- **Дедупликация ответов:** если в `.comments.yaml` есть одинаковые ответы (одинаковый `content`+`dateTime`) — в выводе показан только один. Это встречается при сбоях синхронизации.
- **Рассинхронизация md↔yaml:** если ID есть в yaml но не в md (или наоборот) — вывод помечает это статусом:
  - `anchor_not_found` — ID в yaml, нет тега в md
  - `yaml_missing` — тег в md, нет записи в yaml

## Обработка ошибок

| Ситуация | Поведение |
|----------|-----------|
| Нет `.comments.yaml` рядом с `.md` | Комментариев нет — вывод пустой |
| `.comments.yaml` с битым YAML | Warning в stderr, файл пропускается |
| Путь не существует | Exit 2, ошибка в stderr |
| `<comment id>` без закрывающего тега | Игнорируется (не парсится) — для детекции используй `validate_comments.py` |

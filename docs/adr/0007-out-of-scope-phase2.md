# ADR-0007: Функциональность, перенесённая в Phase 2

**Status:** Accepted
**Date:** 2026-05-08
**Plugin:** gramax

## Context

Spec явно обозначает Out of Scope для MVP. SA-анализ добавляет технические соображения, почему именно эти элементы должны быть отложены, и фиксирует условия для их включения в Phase 2.

## Decision

Следующие элементы исключаются из MVP реализации:

### 1. Полный визуальный SVG-рендер через drawio MCP

**Почему Phase 2:** `drawio_convert.py` генерирует SVG с embedded XML, но без визуального рендера (без нод, стрелок, шрифтов). Полный визуальный рендер требует headless Chromium или drawio desktop — это сложная зависимость, не совместимая с NFR-003/NFR-004 в рамках MVP.

**Условие для Phase 2:** PoC `lgazo/drawio-mcp-server` (Dev-задача): если server предоставляет `render_to_svg(xml)` → valid SVG с геометрией → включить в Phase 2 как optional rendering path.

### 2. AI-генерация drawio XML с нуля из натурального языка (высокий риск)

**Почему Phase 2:** LLM-генерация валидного mxfile XML нестабильна. `jgraph/drawio-mcp` сами рекомендуют Mermaid для надёжного AI-рендеринга — прямое свидетельство сложности. AC-012 (валидный XML) можно покрыть, но качество семантики диаграммы непредсказуемо.

**MVP-поведение:** LLM генерирует mxfile XML; AC-012 проверяет структурную валидность. Качество визуализации — на ответственности пользователя (он редактирует в diagrams.net).

**Условие для Phase 2:** benchmark LLM-генерации по 20+ test cases; acceptance rate ≥ 80% структурно валидных и семантически корректных диаграмм.

### 3. Auto-detection движка

**Почему Phase 2:** Spec явно out-of-scope. Архитектурно: auto-detect требует classifier (LLM или rule-based) с собственным набором edge cases и тестовым покрытием. Usability выигрыш неочевиден — пользователь обычно знает, какой формат хочет.

**Условие для Phase 2:** user research показывает, что явный выбор движка является friction-точкой для >30% пользователей.

### 4. Edit-existing-SVG flow

**Почему Phase 2:** round-trip редактирование (SVG → drawio → update → re-convert) требует дополнительного параметра (`--decompress` в `drawio_convert.py` уже есть) и skill-логики для определения, что файл уже существует и пользователь хочет обновить, а не создать. Это отдельный user journey с собственными AC.

**Условие для Phase 2:** отдельный spec от BA.

### 5. Live-preview mermaid в браузере

**Почему Phase 2:** это функциональность `claude-mermaid` MCP (`mermaid_preview`). Skill не дублирует поведение submodule. Если `mermaid_preview` нужен — пользователь устанавливает `claude-mermaid` плагин.

**Условие для Phase 2:** не применимо (уже реализовано в `claude-mermaid`; интеграция — отдельное решение).

### 6. Batch-генерация нескольких диаграмм за один вызов

**Почему Phase 2:** один вызов = одна диаграмма (FR-001, FR-002). Batch требует цикла, управления ошибками по каждому элементу, atomic-semantics для набора файлов.

**Условие для Phase 2:** явный запрос от пользователей; отдельный spec.

### Специальный случай: позиция вставки ссылки в md

**Текущее MVP-решение (ADR-0005):** вставка в конец файла. Если пользователь указал позицию — LLM выполняет целевую вставку как text-editing задачу.

**Phase 2 (если нужно):** shell-парсер для поиска H2/H3 заголовков и вставки после указанного раздела. Это снизит зависимость от LLM text-editing.

## Consequences

**Положительные:**
- MVP фокусирован и тестируем (12 AC покрыты, edge cases зафиксированы).
- Phase 2 условия явно зафиксированы — не нужно возвращаться к этому решению без новых данных.

**Отрицательные / trade-offs:**
- Пользователи, ожидающие визуально красивых drawio SVG из MVP, получат «плоские» SVG с embedded XML.
- Необходимо явно коммуницировать ограничение в README (Tech-writer задача).

## Alternatives Considered

- **Включить полный визуальный SVG-рендер в MVP** — отклонено: требует headless Chromium или drawio MCP с неподтверждённым headless-рендером; нарушает NFR-003/NFR-004 без PoC.
- **Включить AI-генерацию drawio из нуля без PoC** — отклонено: нестабильность подтверждается авторами `jgraph/drawio-mcp`; AC-012 (структурная валидность) необходимо, но недостаточно для quality gate.
- **Включить auto-detect движка как эксперимент** — отклонено: spec явно запрещает; приводит к неопределённости тестирования.
- **Не фиксировать Phase 2 условия** — отклонено: без явных критериев Phase 2 может никогда не наступить (scope creep prevention).
